หน้าที่ของ ROS Master:
- ROS Master เป็นส่วนกลางในระบบ ROS ที่ทำหน้าที่ควบคุมและจัดการทรัพยากรต่างๆ ในระบบ ROS โดยประกอบด้วยหลายความสามารถดังต่อไปนี้:
  - ทรัพยากรที่จัดเก็บข้อมูลเกี่ยวกับโหนด (nodes) ที่ทำงานอยู่ในระบบ ROS เช่น ชื่อโหนด (node name) และที่อยู่ IP ของโหนด
  - ทรัพยากรที่จัดเก็บข้อมูลเกี่ยวกับโปรแกรมที่ใช้ในโหนด (node) ในระบบ ROS เช่น ที่อยู่ไฟล์ของโปรแกรม (package และ file paths)
  - ทรัพยากรที่จัดเก็บข้อมูลเกี่ยวกับการเชื่อมต่อและการสื่อสารระหว่างโหนด (node) เช่น ที่อยู่ IP และพอร์ตของโหนดที่สื่อสารกัน
  - ทรัพยากรที่จัดเก็บข้อมูลเกี่ยวกับการติดต่อสื่อสารระหว่างโหนดและการสร้างบริการ (service) ในระบบ ROS

การทำงานของ ROS Topics:
- ROS Topics เป็นระบบการสื่อสารแบบแพทเทิร์นใน ROS ที่ใช้ในการส่งข้อมูลระหว่างโหนด (node) ในรูปแบบแบบส่ง-รับ (publish-subscribe) โดยมีลักษณะการทำงานดังนี้:
  - โหนด (publisher) ที่ต้องการส่งข้อมูลจะส่งข้อมูลผ่านทาง Topic ที่กำหนดไว้
  - โหนด (subscriber) ที่ต้องการรับข้อมูลจะสมัครเป็นผู้รับและรอรับข้อมูลจาก Topic นั้นๆ
  - ROS Master จะทราบข้อมูลเกี่ยวกับ Topic และระบุโหนดที่เป็น publisher และ subscriber ของแ

ต่ละ Topic
  - เมื่อโหนด publisher ส่งข้อมูลผ่าน Topic จะมีการส่งข้อมูลไปยังทุกโหนด subscriber ที่สมัครเป็นผู้รับใน Topic นั้นๆ

การทำงานของ ROS Service:
- ROS Service เป็นระบบการสื่อสารแบบแบบบริการ (service-oriented) ใน ROS ที่ใช้ในการส่งคำขอและรับการตอบรับกลับระหว่างโหนด (node) โดยมีลักษณะการทำงานดังนี้:
  - โหนด (service server) ที่ให้บริการจะประกาศและเรียกใช้งานบริการด้วยชื่อที่กำหนดไว้
  - โหนด (service client) ที่ต้องการใช้งานบริการจะเรียกใช้งานบริการด้วยชื่อที่กำหนดไว้
  - ROS Master จะทราบข้อมูลเกี่ยวกับบริการและระบุโหนดที่เป็น service server และ service client ของแต่ละบริการ
  - เมื่อโหนด service client เรียกใช้งานบริการ โหนด service server จะรับคำขอและดำเนินการตามคำขอ จากนั้นจะส่งการตอบรับกลับไปยังโหนด service client

ความแตกต่างระหว่าง Topic กับ message:
- Topic เป็นช่องทางการสื่อสารที่ใช้ในการส่งข้อมูลระหว่างโหนดใน ROS โดยส่ง-รับข้อมูลผ่าน Topic ที่มีชื่อที่กำหนดไว้
- Message เป็นรูปแบบของข้อมูลที่ถูกส่งผ่าน Topic ระหว่างโหนด ซึ่งอาจเป็นข้อมูลที่มีลักษณะต่างๆ เช่น ตัวเลข เลขทศนิยม ข้อความ อาร์เรย์ เป็นต้น

ความแตกต่างในเชิงการใช้งานของ ROS Topic กับ ROS Service:
- ROS Topic ใช้ในการสื่อสารแบบส

่ง-รับข้อมูลแบบแพทเทิร์น ซึ่งโหนด publisher สามารถส่งข้อมูลไปยังหลายโหนด subscriber พร้อมกันได้ และสามารถมีหลายโหนด publisher และ subscriber ใน Topic เดียวกันได้
- ROS Service ใช้ในการสื่อสารแบบบริการ (service-oriented) ซึ่งโหนด service client จะส่งคำขอและรอรับการตอบรับกลับจากโหนด service server โดยมีแค่หนึ่งโหนด service server ให้บริการแก่โหนด service client ในเวลาเดียวกัน
